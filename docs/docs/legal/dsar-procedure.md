# DSAR Procedure

## Intake channel

All privacy and data subject rights requests are received via `privacy@updspace.com`.

## Standard workflow

1. receive request;
2. verify identity and scope of the requester;
3. classify the request: access, export, correction, deletion, restriction, objection, or other;
4. determine which services own relevant data;
5. execute export or erase flows and log the handling path;
6. respond within the applicable legal deadline.

## Technical handling baseline

Because UpdSpace stores user-related data across multiple services, DSAR handling must account for:

- BFF session data;
- Access authorization and audit data;
- Portal profile and membership data;
- Voting participation data;
- Events participation data;
- Gamification grants;
- Activity feed, connectors, and account links.

## Identity verification

Requests must not be fulfilled solely on an unverified email claim if the account or identity context cannot be reliably established.

## Escalation

If the request is legally ambiguous, technically dangerous, or conflicts with fraud-prevention or security duties, the request should be escalated for manual review before execution.
