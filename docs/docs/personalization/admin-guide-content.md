# Admin Guide: Content Management

## Homepage Modals

1. Open `Settings → Personalization`.
2. In Content Management click **Create Modal**.
3. Fill title, content, dates, and type.
4. Use **Preview** to validate RU/EN rendering.
5. Save modal. It appears in table/calendar views.

## Scheduling

- Use `start_date` and `end_date` for controlled visibility.
- Use `display_once` for one-time announcements.
- Use modal order for priority.

## Safe Operations

- Delete is soft-delete by default.
- Restore from bulk action `restore` or restore endpoint.
- Use preview before activation to avoid production mistakes.

## Dashboard Personalization

- Create layout.
- Add widgets from library.
- Set default layout for your user context.
- Toggle widget visibility per layout.

## Analytics

- Open analytics card in personalization settings.
- Track:
  - Views
  - Clicks
  - Dismissals
  - CTR
